# Telegram-бот

Бот работает **в том же процессе**, что и веб-сервер (aiogram polling как asyncio-
задача). Отдельный процесс запускать не нужно. Код: `backend/telegram_runtime.py` +
`backend/telegram_format.py`. Бот, память Horae и LiteLLM — те же, что у веба (общая БД).

## Запуск и настройка

Всё — из веб-админки (🛡 → раздел «Telegram-бот»): токен бота, выбор модели,
`open_to_all`, запуск/остановка (`/api/admin/telegram/start|stop`). Токен можно также
задать в `.env` (`TELEGRAM_BOT_TOKEN`), но управление — из UI.

## Доступ

Два уровня:
- **Белый список по Telegram-ID** (`open_to_all=False`, по умолчанию): пускаем только
  внесённых. Не внесённый получает отказ с подсказкой и своим ID; команда `/request`
  отправляет заявку в админку. Пустой список = никого (важно: пустой ≠ «всем»).
- **`open_to_all=True`** — бот открыт всем.

Проверки делает `_gate(message)` → `(ok, owner_id)`.

## Привязка к аккаунту

Если включён режим аккаунтов, бот работает с **приватными** персонажами/чатами
владельца. Привязка: в веб-приложении 👤 → «Привязать Telegram» → код → в боте
`/link КОД` (`User.telegram_id`). До привязки бот просит привязаться.

## Команды и кнопки

- `/start` — начать; постоянные кнопки: 🎭 Персонажи · 💬 Мои чаты · 🆕 Новый чат ·
  🕘 История · 🧠 Модель · ❓ Помощь.
- `/characters` — выбрать персонажа (инлайн-кнопки, начинает **новый** чат); `/new` —
  новый чат с текущим персонажем; `/info` — сводка по текущему чату; `/model` — выбрать
  модель из прокси; `/friends` — входящие заявки в друзья; `/request` — запросить
  доступ; `/link КОД` — привязать аккаунт.
- **`/chats` — список ВСЕХ своих чатов** (инлайн-кнопки: 🎭 персонаж или 👥 группа ·
  число сообщений, ✅ помечает активный, плюс сниппет последней реплики). Выбор чата
  (`chat:<id>`) делает его активным и **сразу показывает историю** (а для группы —
  ещё и сводку) — можно продолжить любой прежний диалог.
- **`/history` — история текущего чата** (последние ~8 реплик, кто что сказал; в группе
  — с именем каждого персонажа по `speaker_name`).
- **`/info` — сводка по чату**: для группового — участники, сцена, режиссёр вкл/выкл,
  счётчик; для обычного — персонаж и число сообщений.

### Какие чаты видны (Telegram = альтернативный доступ)

Смысл привязки аккаунта — чтобы Telegram давал доступ к тем же диалогам, когда нет
возможности зайти в веб. Поэтому `_own_sessions_filter(tg_user_id, owner_id)`:
- **аккаунт привязан** (`owner_id` есть) → ВСЕ чаты аккаунта по `owner_id`, **включая
  созданные в вебе и групповые**. Можно прямо из Telegram продолжить веб-диалог (новые
  реплики видны и в вебе — сессия одна). `_owns_session` разрешает продолжать любой чат
  владельца;
- **общий режим** (аккаунты выключены, `owner_id=None`) → только чаты, начатые в
  Telegram (`user_key = tg:<id>`) — веб-чаты некому атрибутировать.

### Групповые чаты в боте

Создавать группы в боте нельзя (это делается в вебе), но их можно **открыть, смотреть
историю/сводку и участвовать**. Кто отвечает — та же логика, что в вебе
(`backend/group_chat.py`): по **упоминанию имени** персонажа → **режиссёр** (если включён
`session.director`, ИИ выбирает 1-2) → **по кругу** (round-robin). Генерация —
нестриминговая (`_generate_group_reply` → `group_chat.build_group_messages` + `complete`),
каждая реплика сохраняется с `speaker_name` и уходит **отдельным сообщением с подписью
«🎭 Имя:»**. Единый диспетчер `_respond(message, session_id, text, attachments)` выбирает
групповой или обычный путь для текста, голоса и вложений.

### Активная сессия

Раньше бот всегда продолжал самый свежий чат (`_latest_session` по `id` DESC). Теперь
активный чат хранится в `_active_sessions` (`tg_user_id → session_id`); `/chats`
позволяет переключиться на любой прежний (в т.ч. веб-чат). `_get_or_create_session`
берёт активный (проверив `_owns_session`), иначе — последнюю сессию владельца, иначе
создаёт новую. Создание чата (персонаж/`/new`) сразу делает его активным. Хранилище —
в памяти процесса: после рестарта бота откат на последнюю сессию (сам диалог в БД цел).

## Файлы и большой ввод (один ход к нейросети)

Хэндлеры на `F.text`, `F.voice`, `F.photo`, `F.document`, `F.audio`. Вложения
скачиваются (`_attachment_from_message`) и превращаются в `AttachmentIn`
(картинка/аудио/документ). Документы Word/PDF проходят через `document_service`
(см. [BACKEND.md](BACKEND.md)).

**Альбом** (несколько файлов с подписью) Telegram доставляет отдельными апдейтами с
общим `media_group_id`. `_handle_media` копит их в `_media_buffers` с дебаунсом ~1.3с,
затем `_process_messages` отправляет **все вложения + подписи разом** — одним ходом к
нейросети.

**Длинный текст.** Telegram-клиент режет сообщение > 4096 символов на несколько
апдейтов. `_handle_text` собирает их обратно: короткое одиночное сообщение уходит
сразу (без задержки), но если кусок близок к лимиту (≥ `_SPLIT_HINT` = 4000) или уже
идёт накопление — куски копятся в `_text_buffers` с дебаунсом ~1.5с. `_smart_join`
склеивает их в один ход: продолжение «полного» куска — **встык** (реконструкция
обрезанного клиентом сообщения), отдельные короткие реплики — через пустую строку.
Так модель получает исходное большое сообщение целиком, а не рваные куски.

## Большой ответ (умная разбивка)

Ответ нейросети шлёт `send_long(message, text)`:
1. `telegram_format.split_message` режет текст на части ≤ 4096 символов по границам
   абзацев/строк (в крайнем случае — жёстко), **не разрывая код-блоки** ```` ``` ````
   (нечётный фенс закрывается и переоткрывается в следующей части).
2. Каждая часть конвертируется `markdown_to_html` в Telegram-HTML
   (`<b>/<i>/<s>/<code>/<pre>/<a>/<blockquote>`), чтобы Markdown-разметка реально
   отображалась, а не висела сырыми звёздочками.
3. Шлём с `parse_mode="HTML"`; если разметка не распарсилась
   (`TelegramBadRequest`) — откат на обычный текст, чтобы сообщение точно дошло.

Системный промпт содержит `STYLE_GUIDE` — модель форматирует ответ лёгким Markdown,
который одинаково хорошо смотрится и в приложении, и в Telegram.

## Уведомления

`notify(tg_id, text)` и `notify_friend_request(...)` — сервер шлёт пользователю в
Telegram (например, заявку в друзья с кнопками «Принять/Отклонить»), если бот запущен.

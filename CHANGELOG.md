# История изменений (Changelog)

Формат — по мотивам [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версии — [SemVer](https://semver.org/lang/ru/): МАЖОР.МИНОР.ПАТЧ.
Каждое заметное изменение попадает сюда **в том же коммите**, что и код.

## [1.2.0] — 2026-07-11

### Добавлено
- **Настройка рассуждений (thinking)** — ⚙ → «Генерация» → «Рассуждения 💭»:
  авто / выключены / низкие / средние / высокие. Передаётся в LiteLLM как
  `reasoning_effort` (для Gemini транслируется в `thinkingBudget`); провайдеры
  без поддержки параметр игнорируют.
- **Рассуждения при работе с файлами**: галка «📎 Включать рассуждения при
  работе с файлами» (по умолчанию ВКЛ) — если уровень «авто», а в запросе есть
  вложение (видео/фото/аудио/документ), бюджет размышлений форсируется до
  «средних»: Gemini местами не думает над файлами без явного бюджета.
- **Live-показ размышлений**: пока модель думает, над печатающимся ответом
  виден свёрнутый блок «💭 Модель размышляет…» — можно раскрыть и читать ход
  мысли (новое WS/SSE-событие `thought`; в сохранённый ответ мысли не входят).
- Уровень рассуждений виден в отладочном логе LLM (🐞, поле `reasoning`).

## [1.1.2] — 2026-07-11

### Исправлено
- **Большие файлы падали по таймауту на пути LiteLLM → Vertex.** Таймаут запроса
  к нейросети был фиксированным (120 с), а загрузка большого inline-видео/аудио в
  прокси и Vertex плюс обработка медиа до первого токена занимают дольше. Теперь
  запросы с payload'ом больше 8 МБ получают увеличенный таймаут
  (`LARGE_REQUEST_TIMEOUT`, по умолчанию 600 с; настраивается в `.env`).
- **Сторож стриминга обрывал долгую загрузку файла.** Пока сообщение с крупным
  файлом грузилось на сервер дольше 45 секунд, «сторож» считал генерацию
  зависшей и разблокировал интерфейс прямо во время загрузки. Прогресс загрузки
  теперь обновляет метку активности — сторож не вмешивается.

### Добавлено
- **Прогресс-бар отправки файлов**: сообщение с вложениями уходит через
  XMLHttpRequest с `upload.onprogress` — над композером видна полоса
  «⬆ Отправка на сервер… N% (X из Y МБ)». Работает для чата, артов
  (фото-референсы) и генерации документов в Canvas.

## [1.1.1] — 2026-07-11

### Изменено
- **Убран жёсткий лимит 20 МБ на вложения** (появился в 1.1.0). Сколько реально
  пройдёт — зависит от провайдера и прокси (напрямую в Gemini inline ~20 МБ,
  через Vertex / Files API — заметно больше), поэтому приложение больше не
  решает за провайдера: файл любого размера отправляется как есть, а для очень
  крупных показывается ненавязчивое предупреждение, что при отказе провайдера
  файл стоит сжать.

## [1.1.0] — 2026-07-10

### Исправлено
- **Отправка видео ломала чат.** Видеофайл помечался как «документ», бэкенд
  декодировал бинарник как текст (latin-1 «успешно» читает любые байты) — и в
  запрос к нейросети уходили мегабайты мусора; отправка в чате с таким видео
  падала. Теперь у вложений есть полноценный тип `video`:
  - веб и Telegram-бот (видео, видеосообщения-кружки, документы с mime `video/*`)
    отправляют видео как `inline_data` — Gemini смотрит его нативно;
  - старые сообщения «видео-как-документ» на лету перенаправляются по mime —
    чинить историю чата вручную не нужно;
  - `document_service` больше не читает бинарные файлы как текст (защита
    `_looks_binary`), а честно сообщает модели, что формат не поддержан;
  - вложения больше ~20 МБ отклоняются сразу с понятным сообщением (лимит
    inline-запроса Gemini), а не загадочной ошибкой после отправки;
  - видео проигрывается в пузыре сообщения (`<video controls>`).

### Добавлено
- **Имя и номер чата на виду**: плашка «💬 имя чата · #номер» в шапке,
  всплывающая подсказка с полным именем в списке чатов и групп — теперь легко
  сослаться на конкретный чат («чат #12»).
- **Время сообщений**: у каждого сообщения сохраняется и показывается время
  (🕒 в мете: у вашего — момент отправки, у ИИ — момент готовности ответа);
  полная дата — во всплывающей подсказке. API отдаёт `created_at` (ISO, UTC).
- **Нейросеть видит время пользователя**: перед репликой в контекст добавляется
  системный блок «[Время пользователя] Сейчас у пользователя 14:32, среда
  10.07.2026 (Europe/Moscow)». Часовой пояс — настройка КАЖДОГО чата
  (⚙ → «Персона» → «Часовой пояс этого чата»), по умолчанию определяется по
  браузеру и сохраняется за чатом. Понимает IANA-имена и смещения (`+03:00`).
- **LaTeX-разметка** в сообщениях (KaTeX): `$…$`, `$$…$$`, `\(…\)`, `\[…\]`;
  формулы в код-блоках не трогаются, длинные — со своим горизонтальным скроллом.
- **Запасная модель** (⚙ → «Подключение» → «Запасная модель»): если основная
  не ответила (ошибка провайдера, пустой ответ) — ход автоматически повторяется
  запасной (галочка «Автоматически…»), и/или её можно вызвать вручную кнопкой
  «⚡ запасной моделью» в баннере ошибки. В `model_used` пишется фактическая
  модель ответа; новое WS/SSE-событие `fallback`.
- **CHANGELOG.md** — этот файл: с этой версии история изменений ведётся здесь.

### Изменено
- `GET /api/sessions` и `/api/groups` отдают `timezone`; `PATCH /api/sessions/{id}`
  принимает `timezone`; настройки подключения — поля `fallback_model` и
  `auto_fallback` (см. docs/API.md, docs/DATA_MODEL.md).
- В зависимости добавлен `tzdata` (база часовых поясов для Windows).

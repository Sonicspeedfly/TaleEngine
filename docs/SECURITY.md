# Безопасность и доступ

Настраивается в веб-админке (🛡 → «Безопасность»), хранится в `app_settings.security`
(кэш — `admin_service.security_cache()`). По умолчанию приложение **открыто**.

## Три независимых слоя

### 1. HTTP Basic Auth (внешний барьер)
`BasicAuthMiddleware` (самый внешний middleware). При включении браузер показывает
системное окно входа **ещё до загрузки приложения**. Конфиг:
`security.basic_auth = {enabled, username, password}`. Проверка — `hmac.compare_digest`.
Отдаёт `401 + WWW-Authenticate: Basic` на все пути **кроме** `/ws*` (у WebSocket своя
авторизация, браузерный WS не пройдёт Basic) и `/api/health` (его опрашивает
`start.bat`). Это барьер **поверх** внутреннего входа, а не вместо него.

> После включения нужна перезагрузка страницы — `fetch` не вызывает системное окно,
> его показывает только навигация верхнего уровня; после релоада браузер кэширует
> учётку и шлёт её во всех запросах. `saveSecurity` делает это автоматически.

### 2. Режим аккаунтов (`accounts_enabled`)
Вход по логину/паролю, у каждого свои приватные данные. Первый зарегистрированный —
**администратор**. Авторизация — заголовок `X-User-Token` (таблица `user_tokens`).
Данные скоупятся по `owner_id` (`accounts.scope_query`): пользователь видит своё, админ —
всё. Код доступа в этом режиме не используется.

### 3. Режим кода доступа (по умолчанию)
- `access_code` — код входа в приложение (заголовок `X-Access-Code`); пусто = открыто.
- `admin_password` — пароль администратора (заголовок `X-Admin-Password`) для `/api/admin/*`.

## `AccessMiddleware` — два режима

Гейтит `/api/*` (кроме открытых путей: `/api/health`, `/api/auth/status|login|admin|
register|login_user`):

- **Аккаунты:** `/api/*` требуют валидный `X-User-Token`; `/api/admin/*` — роль `admin`
  (или верный `X-Admin-Password`).
- **Код доступа:** `/api/*` требуют `X-Access-Code` (если задан); `/api/admin/*` —
  `X-Admin-Password` (если задан).

WebSocket авторизуется query-параметром: `?token=` (аккаунты) или `?code=` (код).

## Порядок middleware

Starlette применяет middleware в обратном порядке добавления (добавленный **последним**
— внешний). В `main.py`:
```
add(NoCacheStaticMiddleware)   # внутренний
add(AccessMiddleware)
add(CORSMiddleware)
add(BasicAuthMiddleware)       # внешний — срабатывает первым
```

## Приватность данных

- Персонажи, чаты, персоны, пресеты, **память Horae** скоупятся по владельцу/доступу.
- Память расшаренного чата видна соавтору; глобальный лор — всем; приватная память
  чужого чата и чужой лорбук — скрыты (тест `tests/test_horae_privacy.py`).
- Шаринг чата другу даёт чтение и участие (`SessionShare`); проверка —
  `_can_access_session`.
- Ключ LiteLLM маскируется (`***`) для не-админов; менять подключение может только админ.

## Рекомендации

- Для доступа из интернета используйте **Basic Auth** или обратный прокси с TLS.
- Пароли задавайте надёжные (Basic Auth-пароль хранится в БД в открытом виде —
  это «дверной замок», а не криптозащита).
- `reset_db.bat` стирает всю БД, включая настройки безопасности.

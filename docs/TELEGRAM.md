# Telegram-бот

Бот работает **в том же процессе**, что и веб-сервер (aiogram polling как asyncio-
задача). Отдельный процесс запускать не нужно. Код: `backend/telegram_runtime.py` +
`backend/telegram_format.py`. Бот, память Horae и LiteLLM — те же, что у веба (общая БД).

## Запуск и настройка

Всё — из веб-админки (🛡 → раздел «Telegram-бот»): токен бота, выбор модели,
`open_to_all`, запуск/остановка (`/api/admin/telegram/start|stop`). Токен можно также
задать в `.env` (`TELEGRAM_BOT_TOKEN`), но управление — из UI.

## Доступ

Два уровня:
- **Белый список по Telegram-ID** (`open_to_all=False`, по умолчанию): пускаем только
  внесённых. Не внесённый получает отказ с подсказкой и своим ID; команда `/request`
  отправляет заявку в админку. Пустой список = никого (важно: пустой ≠ «всем»).
- **`open_to_all=True`** — бот открыт всем.

Проверки делает `_gate(message)` → `(ok, owner_id)`.

## Привязка к аккаунту

Если включён режим аккаунтов, бот работает с **приватными** персонажами/чатами
владельца. Привязка: в веб-приложении 👤 → «Привязать Telegram» → код → в боте
`/link КОД` (`User.telegram_id`). До привязки бот просит привязаться.

## Команды и кнопки

- `/start` — начать; постоянные кнопки: 🎭 Персонажи · 🆕 Новый чат · 🧠 Модель · ❓ Помощь.
- `/characters` — выбрать персонажа (инлайн-кнопки); `/new` — новый чат; `/model` —
  выбрать модель из прокси; `/friends` — входящие заявки в друзья (принять/отклонить);
  `/request` — запросить доступ; `/link КОД` — привязать аккаунт.

## Файлы и большой ввод (один ход к нейросети)

Хэндлеры на `F.text`, `F.voice`, `F.photo`, `F.document`, `F.audio`. Вложения
скачиваются (`_attachment_from_message`) и превращаются в `AttachmentIn`
(картинка/аудио/документ). Документы Word/PDF проходят через `document_service`
(см. [BACKEND.md](BACKEND.md)).

**Альбом** (несколько файлов с подписью) Telegram доставляет отдельными апдейтами с
общим `media_group_id`. `_handle_media` копит их в `_media_buffers` с дебаунсом ~1.3с,
затем `_process_messages` отправляет **все вложения + подписи разом** — одним ходом к
нейросети.

## Большой ответ (умная разбивка)

Ответ нейросети шлёт `send_long(message, text)`:
1. `telegram_format.split_message` режет текст на части ≤ 4096 символов по границам
   абзацев/строк (в крайнем случае — жёстко), **не разрывая код-блоки** ```` ``` ````
   (нечётный фенс закрывается и переоткрывается в следующей части).
2. Каждая часть конвертируется `markdown_to_html` в Telegram-HTML
   (`<b>/<i>/<s>/<code>/<pre>/<a>/<blockquote>`), чтобы Markdown-разметка реально
   отображалась, а не висела сырыми звёздочками.
3. Шлём с `parse_mode="HTML"`; если разметка не распарсилась
   (`TelegramBadRequest`) — откат на обычный текст, чтобы сообщение точно дошло.

Системный промпт содержит `STYLE_GUIDE` — модель форматирует ответ лёгким Markdown,
который одинаково хорошо смотрится и в приложении, и в Telegram.

## Уведомления

`notify(tg_id, text)` и `notify_friend_request(...)` — сервер шлёт пользователю в
Telegram (например, заявку в друзья с кнопками «Принять/Отклонить»), если бот запущен.

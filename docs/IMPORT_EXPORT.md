# Импорт и экспорт

В приложении три независимых канала переноса данных:

1. **Нативный формат чата AiChat** — рекомендуемый, с полной точностью (round-trip).
2. **Импорт чатов из SillyTavern** (.jsonl) — для миграции со SillyTavern.
3. **Карточки персонажей SillyTavern** (PNG/JSON V2) — обмен персонажами.

Кнопка импорта чата (💬 в сайдбаре) сама определяет формат файла: если это JSON с
маркером `"format":"aichat.chat"` — нативный импорт; иначе — парсер SillyTavern.

---

## 1. Нативный формат чата (`*.aichat.json`)

> Почему свой формат: экспорт SillyTavern слишком специфичен (его inline-теги Horae,
> служебные сообщения, нюансы свайпов) — под него трудно подстроиться без потерь.
> Нативный формат сохраняет всё, что есть в приложении, и переносит чат между базами
> точно. Код: `backend/native_io.py` + `_import_native_chat` в `backend/main.py`.

- **Экспорт:** `GET /api/sessions/{id}/export` (кнопка 💾 у чата). Доступен владельцу,
  админу и тому, кому чат пошарили.
- **Импорт:** `POST /api/sessions/import` (тот же эндпоинт, что и для SillyTavern —
  формат определяется по маркеру).

### Схема файла

```jsonc
{
  "format": "aichat.chat",
  "version": 1,
  "exported_at": "2026-06-27T12:00:00+00:00",

  "character": {                  // самодостаточно: персонаж воссоздаётся при импорте
    "name": "Аврора",
    "description": "...", "personality": "...", "scenario": "...",
    "first_message": "...", "system_prompt": "...",
    "avatar_path": "data:image/... | URL | null",
    "generation_params": { },     // переопределения параметров генерации
    "model": "имя-модели | null"
  },

  "persona": {                    // опционально (может быть null)
    "name": "...", "description": "...", "avatar_path": "..."
  },

  "session": {
    "title": "...", "scenario": "...", "author_note": "...",
    "background": "...", "is_group": false, "director": false
  },

  "group_members": [ { /* как character */ } ],   // только для групповых чатов

  "messages": [
    {
      "idx": 0,                   // порядковый индекс внутри файла
      "role": "assistant",        // user | assistant | system
      "content": "...",           // активный вариант
      "swipes": ["вариант 1", "вариант 2"],
      "active_swipe": 0,
      "speaker_name": "Аврора",   // кто сказал (для групп) | null
      "reply_to_idx": null,       // ССЫЛКА ПО ИНДЕКСУ, не по id (см. ниже)
      "attachments": [ ],         // как в БД (image/audio/document)
      "model_used": "..."         // чем сгенерировано | null
    }
  ],

  "horae": [
    {
      "scope": "session",         // "session" — память чата; "character" — лорбук персонажа
      "category": "state",
      "title": "...", "content": "...",
      "keywords": [],
      "always_on": true, "enabled": true, "priority": 10
    }
  ]
}
```

### Важные детали реализации

- **`reply_to_idx`, а не id.** Ответы на сообщения хранятся по индексу внутри массива
  `messages`, а не по id из БД. При импорте сообщения создаются, собирается карта
  `idx → новый id`, затем вторым проходом проставляется `reply_to_id`. Так ссылки
  переносятся между базами с разными id.
- **Переиспользование персонажа.** При импорте персонаж ищется по имени в области
  пользователя; если найден — переиспользуется (новый не создаётся).
- **Лорбук персонажа не дублируется.** Записи Horae со `scope:"character"`
  импортируются только если персонаж был создан заново; у существующего персонажа
  лорбук уже есть.
- **Память чата** (`scope:"session"`) импортируется всегда — в новую сессию.
- В режиме аккаунтов всё создаётся с `owner_id` импортирующего пользователя.

## 2. Импорт чатов SillyTavern (`*.jsonl`)

Код: `backend/chat_import.py` (`parse_sillytavern_chat`). Особенности:

- Первая строка — метаданные (`character_name`, `user_name`); если имя — заглушка
  (`unused`/пусто), оно выводится из самих реплик (самое частое имя ассистента).
- **Встроенные теги Horae вырезаются** из текста реплик: `<horae>…</horae>`
  (снимок состояния: `time/location/atmosphere/characters/costume:Имя=описание`) и
  `<horaeevent>…</horaeevent>` (события). Снимки **сливаются** по ключам (поздние
  значения важнее, но детали из ранних блоков сохраняются), события — собираются с
  дедупом. Результат сохраняется как записи памяти: «Состояние сюжета» (always_on) и
  «Хронология событий».
- Системные (`is_system`) и «чисто-тег» реплики в чат как сообщения не попадают, но
  их Horae-данные извлекаются. Поэтому число импортированных **сообщений** = число
  реальных реплик, а служебные снимки состояния уходят в память, а не в диалог.

При импорте создаётся персонаж (или переиспользуется по имени) и новая сессия;
ответ эндпоинта содержит `count`, `horae_saved`, `horae_events_saved`.

## 3. Карточки персонажей SillyTavern

Код: `backend/characters.py`.

- **Импорт** (`POST /api/characters/import`, кнопка 📥): PNG с tEXt-чанком `chara`
  (base64-JSON) или JSON V1/V2. Лорбук (`data.character_book.entries`) превращается в
  записи Horae, привязанные к персонажу (`extract_horae_entries`).
- **Экспорт** (`GET /api/characters/{id}/export`, кнопка ⬇): карточка `chara_card_v2`
  с полями персонажа и `character_book`, собранным из его записей Horae
  (`build_character_book`).
